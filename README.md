full web app from scratch: a Chatbot / Q&A Knowledge Base Platform.

Core Features  
- Admins upload documents / PDFs / markdown articles to build a knowledge base.  
- System processes and indexes the content for semantic search.  
- Users can ask natural-language questions; the chatbot returns best matching answers with links to original documents.  
- Include follow-up / clarification flow (ability for user to ask further questions).  
- Chat history stored per user; ability to review past queries.  
- Authentication: sign up / login; optionally roles (admin vs user).  
- Admin dashboard: manage content, view common questions, see usage analytics.  

UI / UX Requirements  
- Clean, modern design; responsive (desktop + mobile).  
- Sidebar with list of topics/categories; search bar.  
- Chat interface area; ability to show recent articles or suggestions.  
- Dark / light theme switch.  

Technical & Integrations  
- Use an AI model/API for semantic search (e.g. OpenAI embeddings or similar).  
- Database/storage for documents and metadata.  
- REST or GraphQL API endpoints for uploading, querying, user management.  
- Use cloud storage or object storage for document files.  
- Secure authentication (JWT or provider).  

Bonus / Advanced  
- Suggest documents based on past user behavior.  
- Allow users to upvote / flag answers.  
- Versioning of documents; change tracking.  
- Export chat transcripts.  

Ready app: full pages, navigation, data persistence, UI, backend, AI logic.
